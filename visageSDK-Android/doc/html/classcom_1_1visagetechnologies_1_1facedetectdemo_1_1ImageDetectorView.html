Detailed Description

Derives from TrackerView.

Class ImageTrackerView is used for providing detector with necessary image data and controlling surface used to display results of detection. Implements Runnable interface provided by Android. Runnable interface is used for implementation of simple multi-threading.

byte [] com.visagetechnologies.facedetectdemo.ImageDetectorView.ConvertToByte ( )
inline

Utility method called to obtain raw pixel data from bitmap object in which image used for detection is loaded.

Pixel data is obtained as array of integers by calling getPixels method of Bitmap class. Each element of obtained array of integer represents one pixel of bitmap. In order to create pixel data usable by tracker it is necessary to extract red, green and blue color values from array of integers and store it as a byte array.

Returns
byte array of raw pixel data obtained from image used for detection.

void com.visagetechnologies.facedetectdemo.ImageDetectorView.SetOptimalBitmapSize ( )
inline

Utility method used to resize bitmap of image used for tracking.

Images that are larger than the screen are resized to fit the screen while keeping the initial ratio between height and width. In order to interpret images in a correct way OpenCV requires image size to be power of two. Images that have either width or height value that does not satisfy this condition are additionally resized until they do. This slightly changes ratio between width and height.

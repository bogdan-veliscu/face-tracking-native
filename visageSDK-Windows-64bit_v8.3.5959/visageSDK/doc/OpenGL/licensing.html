This page focuses on technical issues such as obtaining and using the license key files. For detailed commercial conditions please contact Visage Technologies.

Applications developed using visage|SDK must be licensed in order to function. Licensing is based on License Key Files. A single License Key File is valid for one or more installations on one or more platforms (operating systems). The License Key File can be opened in a text editor (e.g. Windows Notepad or similar) in order to read the licensing information, but it is important not to change the file in any way as it would become invalid.

Applications without a valid license key will still function, but tracking time shall be limited to one minute and appropriate warning messages shall be displayed.

Obtaining the License Key File

For 30-day evaluation, simply send an email request to your Visage Technologies contact person and you will receive a License Key File by email.

If you are purchasing a license, your Visage Technologies contact person will deliver your License Key File by email based on your purchase order.

Using the License Key File with Sample projects

visage|SDK Sample projects already contain the necessary licensing initialization code so they can be built and used without any modifications.

Place the License Key File into the visageSDK/bin folder. The license will be automatically registered on first use of the built Sample application (network connection is required at this time).

Using the License Key File in own projects

The License Key File must be distributed with your Application and be accessible to it. In your application, you need to call the licensing initialization function in order to let the Application read the License Key File. This is done as follows in your application code:


a) C++ API

namespace VisageSDK //all visage|SDK calls must be in visageSDK namespace
{
  // declaration to be placed in your .cpp or .h file
  void __declspec(dllimport) initializeLicenseManager(const char *licenseKeyFileFolder);

  void main() // or equivalent function that executes when your Application starts
  {
    initializeLicenseManager("path");
    // ... the rest of your application
  }
}

b) C# API

using VisageWrapper;

namespace MyNamespace
{
  class MyClass
  {
    void main() // or equivalent function that executes when your Application starts
    {
      VisageLicensing.initializeLicenseManager("path");
      // ... the rest of your application
    }
  }
}

"path" is the folder that contains the License Key File. The simplest is to use "." as the path, and place the License Key File into the same folder with the Application.

The license will be automatically registered with the Visage Technologies Licensing Server on first use of the Application, so a working network connection is required at this time. When the License Key File is valid for multiple installations, the Licensing Server allows the licensed number of installations. Some license types (e.g. floating licenses) make automatic connection to the Licensing Server on every use of the Application.

If multiple Windows users use the Application, network connection to the licensing server is performed for each user at first use of the Application; this can be avoided using manual registration process - see next section.

Manual registration

While the described automatic registration of licenses is usually the most convenient way to use Visage Technologies licensing, it is also possible to register the license manually using the Registration Manager application (registrationmanager.exe found in the Visage Technologies folder).

IMPORTANT: you must have Administrator privileges to run the Registration Manager - use the "Run as Administrator" option (your account must have administrator privileges).

To register the license manually, please copy the License Key File into the Visage Technologies folder, then start the Registration Manager . The Registration Manager will recognize the license, display a dialog box with its information and register it. Registration Manager will connect to the Licensing Server so network connection is required.

NOTE: after performing the manual registration, it is still necessary to follow the instructions in the section "Using the License Key File in own projects" in order to let your application access the License Key File.

Registration without network connection

Alternatively, for users wishing to avoid networked registration, it is possible to perform a fully manual registration in the following way:

  • Run Registration Manager before requesting the Licence Key File.
  • Copy the 12-digit Device ID displayed by the Registration Manager and send it by email to your Visage Technologies contact person when requesting the License Key File.
  • You will receive a device-specific License Key File that does not require registration with the Licensing Server.
  • Copy the License Key File you received into the Visage Technologies folder, then run the Registration Manager again. It will recognise the License Key, provide you the information about the License Key you obtained, and register it locally on your computer (no network connection required).
  • Follow the instructions in the section "Using the License Key File in own projects" in order to let your application access the License Key File.

Removing the license or moving it to another computer

To remove the previously registered license from your computer:

  • Start the Registration Manager (see previous section).
  • Select the license you want to remove from the displayed list of registered licenses, then click on "Remove license".
  • After an additional confirmation, the license is removed.
  • A message containing the removal details is displayed; copy this message and send it by email to licensing@visagetechnologies.com.
  • You will receive a confirmation by email, normally within one business day.
  • Additionally, for refference purposes, the removal information is stored in the file named "license-removal-xxx.txt in the Visage Technologies folder.

After the removal is confirmed by Visage Technologies, you can use the original License Key File to register the license on another computer.

Web License

Web License allows deploying one Web Application on a single server. The license registration technique depends on whether your Web Application is server-based (runs on the server) or client-based (runs on the end-user's computer, typically as a browser plugin or similar). Combined implementation (both client- and server-side) is possible by simply using both registration procedures explained below.


Server-based Web Applications

The procedure for obtaining and registering server-side licenses is identical to the process explained above.


Client-based Web Applications

With this type of Web license you have received the License Key File registered for a specific URL where your application will be deployed. The License Key File must be distributed together with the Application.

You need to call the initialization function in order to let the Application read it. This is done as follows in your application code:

namespace VisageSDK //all visage|SDK calls must be in visageSDK namespace
{
  // declaration to be placed in your .cpp or .h file
  void __declspec(dllimport) initializeLicenseManager(const char *licenseKeyFileFolder);

  void main() // or equivalent function that executes when your Application starts
  {
    initializeLicenseManager("path");
    // ... the rest of your application
  }
}

"path" is the folder that contains the License Key File. The simplest is to use "." as the path, and place the License Key File into the same folder with the Application.
